# AI Continuity Methodology v1.0

## Purpose
This repository is not written only for humans.
It is written so that future AI systems can continue the research without losing reasoning context.

## Core Principle
Chats are temporary.
Git is persistent.
Reasoning must be preserved, not only conclusions.

## Every important research document must contain:
1. Context
2. Original discussion and reasoning
3. Competing ideas
4. Turning points
5. Final decision
6. Open questions
7. Context for future AI collaborators
8. Next actions

## Rule
Never summarize away important reasoning.
Capture why decisions were made, including rejected alternatives.

## Goal
Create a Human + AI collective research memory that survives changes in AI models and enables future collaboration across different AI systems.
